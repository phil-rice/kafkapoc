1. **CEL is evaluated AFTER enrichment**
   - Product category must be resolved first
   - Location lookups must be complete
   - Contact info must be extracted

2. **Outputs are conditional**
   - Not all events generate notifications
   - All events generate tracking
   - Only ENKDN generates billing

   - No contact = no notification
   - Email only = email notifications only
   - Both = up to 2 notifications per rule

4. **Rules are deterministic**
   - Same input always produces same output
   - No randomness or external API calls
   - Easy to test and validate
